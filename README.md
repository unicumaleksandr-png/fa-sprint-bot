# FA Career Bot — GitHub Actions Edition
### Автономный Telegram-бот для 30-дневного спринта | Big4 → Financial Analyst

Бот работает **полностью на GitHub** — никакого сервера, никакого постоянно включённого компьютера.
GitHub Actions запускает скрипты по расписанию и обрабатывает твои команды каждые 30 минут.

---

## Как это работает

```
GitHub Actions (бесплатно, облако)
       │
       ├── 07:00 МСК → утренний план дня
       ├── 09:25 МСК → напоминание (конец первого блока)
       ├── 14:00 МСК → дневной импульс
       ├── 21:00 МСК → вечерний чек-ин
       ├── каждые 30 мин → слушает твои команды (/done, /skip...)
       └── воскресенье 20:00 МСК → еженедельный разбор

Прогресс (state/progress.json) хранится прямо в репозитории.
Бот коммитит его обратно после каждого обновления.
```

## Быстрый старт на Mac — 4 шага, ~10 минут

### Шаг 1 — Создай Telegram-бота

1. Открой Telegram → найди **@BotFather** → напиши `/newbot`
2. Придумай название (например: `FA Sprint Aleksandr`)
3. Придумай username (например: `fa_sprint_aleks_bot`)
4. Скопируй **токен** — выглядит как `7123456789:AABBccDDee...`

Узнай свой Telegram ID:
- Найди **@userinfobot** → напиши любое сообщение
- Он ответит твоим `Id:` — это и есть `CHAT_ID`

---

### Шаг 2 — Создай репозиторий на GitHub

1. Открой [github.com/new](https://github.com/new)
2. Название: `fa-sprint-bot` (или любое другое)
3. Видимость: **Public** ← важно для бесплатных минут Actions
4. Нажми **Create repository**

Загрузи файлы (через терминал на Mac):

```bash
# Установи Git если нет (откроет установщик Xcode Tools)
git --version

# Клонируй пустой репозиторий
git clone https://github.com/ВАШ_ЛОГИН/fa-sprint-bot.git
cd fa-sprint-bot

# Скопируй все файлы бота в папку fa-sprint-bot
# (перетащи содержимое папки fa_bot_github сюда)

# Настрой дату старта спринта в state/progress.json
# "start_date": "2026-04-23"  ← поменяй на сегодняшнюю дату

# Первый коммит
git add .
git commit -m "init: FA sprint bot"
git push origin main
```

---

### Шаг 3 — Добавь секреты в GitHub

1. Открой репозиторий на GitHub
2. **Settings** → **Secrets and variables** → **Actions** → **New repository secret**
3. Добавь два секрета:

| Name        | Value                          |
|-------------|--------------------------------|
| `BOT_TOKEN` | `7123456789:AABBccDDee...`     |
| `CHAT_ID`   | `123456789`                    |

> Секреты **никогда** не попадают в код репозитория — это безопасно даже для публичного репо.

---

### Шаг 4 — Активируй Actions и проверь

1. Открой вкладку **Actions** в репозитории
2. GitHub может попросить разрешить запуск workflows → нажми **Enable**
3. Проверь что всё работает: нажми на workflow **«☀️ Утренний план»** → **Run workflow** → **Run**
4. Через 30 секунд в Telegram придёт утреннее сообщение 🎉

## Расписание сообщений

| Время (МСК) | Workflow                  | Что происходит                         |
|-------------|---------------------------|----------------------------------------|
| 07:00       | `morning.yml`             | Утренний план с кнопками задач         |
| 09:25       | `reminders.yml`           | Напоминание: конец первого блока       |
| 14:00       | `reminders.yml`           | Дневной импульс: Excel и английский    |
| 21:00       | `evening.yml`             | Вечерний чек-ин                        |
| каждые 30 м | `interactive.yml`         | Слушает /done, /skip и другие команды  |
| Вс 20:00    | `weekly.yml`              | Еженедельный разбор + обновление стрика|

## Команды бота

| Команда              | Действие                             |
|----------------------|--------------------------------------|
| `/start`             | Приветствие и список команд          |
| `/plan`              | Показать план на сегодня             |
| `/ready`             | Подтвердить старт рабочего дня       |
| `/done fa`           | Отметить задачу выполненной          |
| `/skip sport лень`   | Пропустить задачу с причиной         |
| `/report`            | Статус задач сегодня                 |
| `/week`              | Итоги текущей недели                 |
| `/progress`          | Общий прогресс спринта               |
| `/motivation`        | Мотивационное сообщение              |

**Задачи:** `fa` · `ifrs` · `excel` · `english` · `sport`

**Быстрые кнопки:** утреннее и вечернее сообщения содержат кнопки — нажимаешь и задача засчитана.

**Отчёт цифрой:** в ответ на вечерний чек-ин напиши просто `4` (сколько задач выполнено из 5).

## Как хранится прогресс

Всё состояние хранится в `state/progress.json` прямо в репозитории.
После каждого обновления бот делает git commit и push.

```json
{
  "start_date": "2026-04-23",
  "name": "Александр",
  "current_streak": 7,
  "best_streak": 7,
  "total_days": 10,
  "logs": {
    "2026-04-23": {
      "fa_done": true,
      "ifrs_done": true,
      "excel_done": true,
      "english_done": true,
      "sport_done": false,
      "total_done": 4,
      "skipped": ["sport:усталость"]
    }
  }
}
```

Ты всегда можешь открыть этот файл в GitHub и увидеть весь свой прогресс.

## Как изменить время напоминаний

Все расписания — в файлах `.github/workflows/*.yml`, строка `cron`.
Формат: `минуты часы * * *` (в UTC, МСК = UTC+3).

Примеры:
```yaml
- cron: "0 4 * * *"    # 07:00 МСК
- cron: "25 6 * * *"   # 09:25 МСК
- cron: "0 11 * * *"   # 14:00 МСК
- cron: "0 18 * * *"   # 21:00 МСК
```

Изменил файл → сделал `git push` → новое расписание работает автоматически.

## Лимиты GitHub Actions

| Тип репозитория | Минуты в месяц | Рекомендация                         |
|-----------------|----------------|--------------------------------------|
| **Публичный**   | **Безлимит**   | Используй — это бесплатно            |
| Приватный       | 2 000 мин      | В `interactive.yml` замени `*/30` на `0 *` (раз в час) |

## Ручной запуск любого workflow

**Actions** → выбери workflow → **Run workflow** → **Run workflow**

Полезно:
- Запустить утренний план вне расписания
- Немедленно обработать команду (не ждать 30 минут)
- Проверить, что всё работает после настройки

---

## Локальный запуск на Mac (для тестирования)

```bash
# Установи Python 3.11+ если нет
brew install python@3.11   # или скачай с python.org

# В папке проекта
pip3 install -r requirements.txt

# Создай .env файл
cp .env.example .env
# Заполни BOT_TOKEN и CHAT_ID

# Загрузи переменные и запусти любой скрипт
export $(cat .env | xargs) && python3 bot/send_morning.py

# Или интерактивный режим (слушает 60 секунд)
export $(cat .env | xargs) && RUN_SECONDS=60 python3 bot/interactive.py
```
